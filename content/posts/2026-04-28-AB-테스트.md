요즘 쇼핑몰 매출이 정체되어서 뭔가 돌파구가 필요했다. 광고비는 계속 올라가는데 전환율은 그대로니까 수익성이 점점 나빠지더라.

**결론부터 말하면, 3개월간 체계적으로 A/B 테스트를 돌린 결과 매출이 32% 올랐다. 전환율은 2.1%에서 2.8%로 증가했고, 평균 주문액도 15% 늘었다.**

## 왜 A/B 테스트를 시작했나

**A/B 테스트는 추측 대신 데이터로 의사결정을 내리는 방법이다.**

솔직히 예전엔 "이게 더 좋아 보이니까 바꿔보자" 식으로 감으로만 웹사이트를 수정했다. 그런데 바꿀 때마다 매출이 오를지 떨어질지 도박 같았어. 특히 메인페이지 레이아웃을 바꿨다가 매출이 20% 떨어져서 급하게 롤백한 경험이 있었다.

그때 깨달았다. 더 이상 감으로 할 순 없다고.

## 첫 번째 테스트: 상품 상세페이지 CTA 버튼

가장 먼저 상품 상세페이지의 "장바구니 담기" 버튼을 테스트했다.

- **A안**: 기존 파란색 버튼 "장바구니 담기"
- **B안**: 주황색 버튼 "지금 구매하기"

코드앤버터로 설정하는 게 생각보다 간단했다. 페이지 커스터마이징 기능으로 기존 버튼 색깔이랑 텍스트를 바꾸고, 트래픽을 50:50으로 분할 설정했다.

3주 후 결과:
- A안 전환율: 2.1%
- **B안 전환율: 2.6% (24% 증가)**

주황색이고 "지금 구매하기"라는 직접적인 문구가 더 효과적이었다. 이 하나만으로도 일 매출이 평균 8만 원씩 늘었어.

## 두 번째 테스트: 이탈 방지 팝업 타이밍

**이탈 방지 팝업은 타이밍이 생명이다.**

처음엔 마우스가 화면 밖으로 나가려 할 때(exit intent) 팝업을 띄웠는데, 모바일에선 이 기능이 잘 안 된다. 그래서 두 가지로 나눠서 테스트했다.

- **A안**: Exit intent + 30초 후 자동 팝업
- **B안**: 스크롤 50% 후 + 60초 후 자동 팝업

B안이 압승이었다. 이탈율이 67%에서 52%로 떨어졌고, 팝업 전환율도 B안이 3.2%로 A안(1.8%)보다 높았다.

팝업 내용은 "첫 구매 10% 할인 + 무료배송"으로 동일하게 했는데, 타이밍만 바꿨는데도 결과가 확연히 달랐어.

## 세 번째 테스트: 상품 리스트 페이지 레이아웃

이게 제일 큰 임팩트가 있었다.

- **A안**: 기존 4열 그리드, 상품명-가격-리뷰 순서
- **B안**: 3열 그리드, 가격-할인율-상품명 순서, 썸네일 크기 20% 확대

B안이 클릭률을 18% 올렸다. 상품이 더 크게 보이니까 시각적 임팩트가 확실히 달랐어. 가격을 먼저 보여주는 것도 좋았던 것 같다.

## 실제 설정 과정 (코드앤버터 기준)

1. **테스트 생성**: 대시보드에서 "새 A/B 테스트" 클릭
2. **페이지 설정**: 테스트할 페이지 URL 입력 (와일드카드 지원)
3. **변형 만들기**: B안 디자인을 페이지 커스터마이징으로 수정
4. **트래픽 분할**: 50:50으로 설정 (처음엔 90:10으로 신중하게 시작해도 됨)
5. **목표 설정**: 전환 이벤트 정의 (구매 완료, 회원가입 등)
6. **통계 설정**: 신뢰도 95%, 최소 샘플 사이즈 500 설정

생각보다 복잡하지 않았다. 코드 건드릴 필요도 없고, 클릭 몇 번이면 끝이더라.

## 3개월 총 결과

- **매출**: 32% 증가 (월 평균 1,200만 원 → 1,580만 원)
- **전환율**: 2.1% → 2.8% (33% 향상)
- **평균 주문액**: 68,000원 → 78,000원 (15% 증가)
- **이탈율**: 67% → 52% (22% 개선)

가장 큰 변화는 마인드셋이었다. 이제 뭔가 바꿀 때마다 "테스트해보자"가 기본 생각이 됐어. 실패하는 테스트도 있지만, 실패해도 배우는 게 있으니까 손해는 아니다.

## 아쉬웠던 점

모바일 최적화 테스트를 더 일찍 했어야 했다. 우리 트래픽의 70%가 모바일인데, PC 위주로만 테스트하다가 시간을 좀 낭비했어.

그리고 테스트 기간을 좀 더 길게 잡을걸 싶다. 2주차에 결과가 좋다고 성급하게 적용했다가, 나중에 보니까 통계적으로 애매한 경우가 있었거든.

다음엔 장기 고객 생애가치(LTV)까지 추적해서 테스트해볼 예정이다. 단순히 전환율만 높이는 게 아니라, 리텐션까지 고려한 최적화를 해보고 싶어.

## 자주 묻는 질문

**A/B 테스트 결과가 나오는데 얼마나 걸리나요?**

보통 2-4주 정도 걸린다. 트래픽이 많으면 더 빨리 나오고, 적으면 좀 더 기다려야 해. 통계적 유의성 95% 달성이 기준이다.

**A/B 테스트할 때 가장 중요한 건 뭔가요?**

한 번에 하나씩만 테스트하는 거다. 여러 개를 동시에 바꾸면 뭐가 효과가 있었는지 모른다. 그리고 충분한 샘플 사이즈를 확보해야 해.

**A/B 테스트 도구 없이도 할 수 있나요?**

수동으로도 가능하지만 정말 번거롭다. 트래픽 분할, 결과 측정, 통계 분석까지 다 수동으로 해야 해서 실수할 확률이 높다.

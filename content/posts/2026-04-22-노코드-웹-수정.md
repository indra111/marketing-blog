# 코딩 모르는 쇼핑몰 사장이 웹사이트 직접 수정하는 꿀팁 5가지

우리 쇼핑몰 메인페이지가 너무 구려서 고객들이 들어오자마자 나가더라. 개발자 부르면 비용도 많이 들고 시간도 오래 걸리고... 그래서 직접 해보기로 했다.

**결론부터 말하면, 코딩을 전혀 모르는 사람도 웹사이트를 충분히 수정할 수 있다. 다만 올바른 방법과 도구만 알면 된다.**

## 1. 일단 백업부터 하기

**웹사이트 수정 전에는 반드시 백업을 해둬야 한다.**

이거 정말 중요하다. 나도 처음에 백업 안 하고 막 건드렸다가 사이트가 완전히 깨져서 식은땀 났던 기억이 있다. 

대부분의 웹호스팅 업체에서 백업 기능을 제공한다. 카페24면 관리자 페이지에서 "백업/복원" 메뉴 찾으면 되고, 워드프레스면 UpdraftPlus 같은 플러그인 쓰면 된다. 

백업은 파일 백업과 데이터베이스 백업 둘 다 해야 한다. 귀찮아도 꼭 하자.

## 2. 워드프레스 페이지 빌더 활용하기

**워드프레스를 쓴다면 페이지 빌더가 가장 쉬운 방법이다.**

Elementor, Divi, Beaver Builder 같은 도구들이 있는데, 이중에서 Elementor를 추천한다. 무료 버전만으로도 충분히 쓸 만하다.

사용법은 간단하다. 위젯을 드래그해서 페이지에 떨어뜨리고, 텍스트나 이미지를 바꾸면 끝이다. 실제로 써보니까 파워포인트 만지는 느낌이더라.

다만 처음엔 어떤 위젯을 써야 할지 모르겠더라. 이럴 땐 템플릿을 가져다 쓰고 내 내용으로 바꾸는 게 낫다.

## 3. CSS 한 줄로 디자인 바꾸기

**간단한 CSS 몇 줄만 알아도 웹사이트 분위기를 확 바꿀 수 있다.**

코딩이라고 하니까 어려워 보이지만, 색깔 바꾸고 폰트 크기 조절하는 정도는 누구나 할 수 있다.

예를 들어:
- 버튼 색깔 바꾸기: `background-color: #ff6b6b;`
- 글자 크기 키우기: `font-size: 18px;`
- 여백 조정하기: `margin: 20px;`

이런 것들은 구글에 "CSS 버튼 색깔 바꾸기" 이렇게 검색하면 다 나온다. 복붙해서 쓰면 된다.

워드프레스면 "외모 > 사용자 정의하기 > 추가 CSS"에서 넣으면 되고, 다른 플랫폼이라도 보통 CSS 추가하는 메뉴가 있다.

## 4. 웹사이트 빌더 도구 활용하기

**완전 초보라면 위그싱, 스퀘어스페이스 같은 웹사이트 빌더를 쓰는 게 낫다.**

이런 도구들은 정말 블록 쌓는 것처럼 웹페이지를 만들 수 있다. 템플릿도 예쁜 게 많아서 그냥 내 콘텐츠만 바꿔도 그럴듯하다.

단점은 기존 사이트를 갈아엎어야 한다는 것. 그리고 매월 이용료를 내야 한다. 하지만 개발자 고용하는 것보다는 훨씬 저렴하다.

나는 테스트용으로 위그싱에서 랜딩페이지 하나 만들어봤는데, 1시간 만에 뚝딱 나오더라. 물론 완성도는 개발자가 만든 것보다 떨어지지만, 급할 때 쓰기엔 충분했다.

## 5. 브라우저 개발자 도구로 실시간 미리보기

**브라우저 개발자 도구를 쓰면 실제로 적용하기 전에 미리 볼 수 있다.**

크롬에서 F12 누르면 개발자 도구가 뜨는데, 여기서 HTML이나 CSS를 임시로 수정해볼 수 있다. 실제 사이트에 반영되지 않고 내 브라우저에서만 보이는 거라서 안전하다.

예를 들어 버튼 색깔을 바꾸고 싶으면, 버튼에 마우스 오른쪽 클릭해서 "검사"를 누르고, CSS 부분에서 색깔 코드를 바꿔보면 된다. 마음에 들면 그때 실제 사이트에 적용하면 된다.

처음엔 복잡해 보이지만, 몇 번 만져보면 금방 익숙해진다. 이거 알고 나면 웹사이트 수정이 훨씬 수월해진다.

## 자주 묻는 질문

**코딩을 전혀 모르는데 웹사이트 수정이 가능한가요?**
가능하다. 요즘엔 노코드 도구들이 많아서 드래그앤드롭만으로도 웹페이지를 수정할 수 있다. 다만 복잡한 기능은 여전히 개발자가 필요하다.

**웹사이트 수정할 때 가장 먼저 해야 할 일은 무엇인가요?**
백업부터 해야 한다. 뭔가 잘못돼도 원래대로 되돌릴 수 있어야 하니까. 그리고 현재 사이트의 문제점을 먼저 파악하는 게 중요하다.

**노코드 웹 수정 도구 중에서 초보자에게 가장 쉬운 건 뭔가요?**
워드프레스 페이지 빌더나 위그싱 같은 도구가 직관적이다. 블록을 조립하는 느낌으로 웹페이지를 만들 수 있어서 처음 써보는 사람도 금방 익힌다.

솔직히 처음엔 "내가 이런 걸 할 수 있을까?" 싶었는데, 막상 해보니까 생각보다 어렵지 않더라. 물론 전문 개발자만큼은 못 하지만, 간단한 수정 정도는 충분히 혼자 할 수 있다.

가장 중요한 건 일단 시작해보는 거다. 백업만 확실히 해두면 뭘 잘못해도 되돌릴 수 있으니까 겁낼 필요 없다. 다음엔 좀 더 고급 기능들도 도전해볼 예정이다.
